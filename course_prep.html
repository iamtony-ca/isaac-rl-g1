Isaac Sim & 휴머노이드 로봇 기초

articulation — 로봇이 무엇으로 이뤄졌고 RL로 어떻게 이어지는가

모든 것(바닥·조명·로봇)은 USD Stage의 Prim 트리. 로봇 = 링크(강체)들이 조인트로 연결된 articulation. 직접 검사 스크립트를 돌려 실측했다.

#관절 이름 = RL 설정의 "주소"

Isaac Lab의 모든 reward/observation은 관절·링크를 정규식 이름으로 지정한다. 이걸 모르면 4일차 튜닝을 못 한다.

".*_hip_pitch_joint"   # 좌/우 고관절 pitch 동시 매칭
".*_knee_joint"        # 좌/우 무릎
".*_ankle_roll_link"   # 발목 링크 (발–지면 접촉 판정)
"torso_joint"          # 몸통 (단수)
⚠ 실측에서 드러난 함정 관절 인덱스 순서는 설정 그룹(legs→feet→arms) 순이 아니라 키네마틱 트리 순회 순서로 뒤섞여 있다 ([0]left_hip_pitch [1]right_hip_pitch [2]torso_joint …). 그래서 action/observation을 숫자 인덱스로 하드코딩하면 안 되고 항상 이름/정규식으로 지정한다.

#Articulation → RL 연결 (실측)

  • Action = 37개 관절의 목표각 offset (×0.5, 기본자세 기준). 직접 토크가 아니다.
  • Observation = base 선/각속도 + projected_gravity(균형의 열쇠) + joint pos/vel + last action.
  • 대부분 관찰에 노이즈를 일부러 주입 → sim-to-real 강건성(5·6일차 복선).
자기점검
  1. USD Prim / articulation root / link / joint / DOF를 각각 한 줄로 설명할 수 있는가?
  2. G1의 action은 토크인가 각도인가? action 차원은 무엇과 같은가?
  3. 왜 관절을 숫자 인덱스 대신 정규식 이름으로 지정하는가?

#액추에이터 = PD 게인 (튜닝 감각의 핵심)

torque = Kp·(q*−q) − Kd·q̇ (effort_limit로 clip). 실측 G1 그룹별 배분:

그룹관절KpKd왜 이렇게
legs9≈1785체중 지탱·추진 → 강하게
feet4202발목은 부드럽게 → 충격 흡수
arms244010자세 유지 → 감쇠 크게
직관무게 받치는 관절은 딱딱하게, 지면에 부딪히는 발목은 물렁하게, 균형용 팔은 감쇠 크게 — 이 배분 자체가 설계 지식이다. action scale(0.5)과 Kp는 함께 "정책이 얼마나 세게 움직일 수 있나"를 결정한다.

#Domain Randomization = sim-to-real의 시작

  • 발–지면 마찰, 링크 질량 / 무게중심, 외부 밀기를 학습 중 무작위화.
  • 시뮬에서 일부러 "불확실성"을 학습해야 실제 로봇(마찰·질량·지연이 다름)에서도 동작한다.

Flat G1은 obs 123 → action 37의 MDP. rsl_rl의 실제 하이퍼파라미터로 PPO 5단계를 본다.

보행 · Reward 설계 / 튜닝

강좌 하이라이트 — weight 하나로 걸음이 무너진다

reward = 여러 항의 가중합. 세 부류로 나눠 균형을 잡는 것이 설계의 전부.

① Task +

"명령대로 움직여라"

track_lin_vel · track_ang_vel · feet_air_time

② Safety − 큼

"넘어지지 마라"

termination_penalty (−200)

③ Style − 작음

"자연스럽고 효율적으로"

action_rate · dof_acc · flat_orientation · feet_slide · joint_deviation · dof_torques …

exp 커널을 왜 쓰나 track = exp(−error²/std²) — 목표 근처에서 기울기가 완만해 "거의 맞았을 때"도 꾸준히 보상 → 미세 추종이 안정적. feet_air_time은 한 발씩(single stance) 체공을 보상해 번갈아 걷기를 유도한다.
